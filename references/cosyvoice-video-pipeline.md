# CosyVoice Course Video Pipeline

Use this reference when the user asks to turn a lecture HTML, speaker script, and local CosyVoice voice into a finished course video.

## Local Defaults

- CosyVoice project: `D:\CosyVoice`
- Work folder: `D:\CosyVoice\work`
- Voice prompt: `D:\CosyVoice\work\congge_prompt_16k.wav`
- Voice instruction: `用沉稳低沉、富有磁性和感染力的语气，沉着有力、略带激情但克制地娓娓道来<|endofprompt|>`
- Model: `D:\CosyVoice\pretrained_models\CosyVoice2-0.5B`
- Preferred output folder for AI Course videos: `C:\Users\arnol\iCloudDrive\AI course\AI Course\乘势课程\视频流水线`

Do not overwrite files under `D:\CosyVoice` unless the user asks. Generate new lesson-specific work folders under `D:\CosyVoice\work\<lesson_id>`.

## Input Contract

Prepare these assets before composing a finished video:

1. A lecture HTML deck, usually generated by this skill and already visually verified.
2. A speaker script split into slide-level segments.
3. Optional software-operation video clips, already trimmed or marked with start/duration.
4. A config JSON for `scripts/course_video_pipeline.py`.

The segment list is the source of truth. Use:

```json
{
  "type": "slide",
  "slide": 1,
  "text": "这一页的口播稿。"
}
```

For a software demo insert:

```json
{
  "type": "demo",
  "path": "C:/Users/arnol/Videos/demo.mp4",
  "start": 0,
  "duration": 120
}
```

If the demo clip already has confirmed narration and timing, keep its original audio. If the user wants AI narration over the demo, create a separate demo-specific script and compose it with FFmpeg first, then insert the resulting clip.

## Commands

Create a starter config:

```powershell
python scripts/course_video_pipeline.py init-config --config C:\path\to\lesson_config.json
```

Render slide screenshots:

```powershell
python scripts/course_video_pipeline.py shots --config C:\path\to\lesson_config.json
```

Generate local CosyVoice audio and apply magnetic processing:

```powershell
python scripts/course_video_pipeline.py voice --config C:\path\to\lesson_config.json
```

Compose final video:

```powershell
python scripts/course_video_pipeline.py compose --config C:\path\to\lesson_config.json
```

Run a quick media/QC pass:

```powershell
python scripts/course_video_pipeline.py qc --config C:\path\to\lesson_config.json
```

Run everything:

```powershell
python scripts/course_video_pipeline.py all --config C:\path\to\lesson_config.json
```

Use `--force` only when rebuilding all cached screenshots, audio, or clips is intended.

## Quality Rules

- Keep voice generation segmented. Prefer 30-60 second chunks; do not generate a whole lesson in one shot.
- Name raw audio `seg_XX.wav`, processed audio `seg_XX_mag.wav`, slide screenshots `slide_XX.png`, and clips `part_XX_*.mp4`.
- After audio generation, listen to at least the first segment and one middle segment before composing.
- After compose, run `qc` and inspect generated PNGs for blank frames, wrong slide order, distorted demo video, and bad ending.
- Copy final MP4 to both the course video folder and any requested Codex `outputs` folder when the user wants a local deliverable.

## 5.2 Proven Pattern

The 5.2 WorkBuddy finished lesson used this structure:

1. Render 16 HTML slides to 1920x1080 PNG.
2. Generate slide narration locally with CosyVoice2 and Congge prompt voice.
3. Apply the magnetic FFmpeg filter chain.
4. Compose slides 1-11.
5. Insert the confirmed WorkBuddy software-operation clip.
6. Compose slides 12-16.
7. End with the corrected message: the goal is not just fewer missed deals, but building an AI-native sales organization.

Use this as the default pattern for future practical lessons: concept slides first, real software operation in the middle, business-level summary at the end.
